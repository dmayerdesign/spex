# spex: agent workflows for spec-as-source projects

The emerging practice of spec-driven development, and its most hardcore variation, "spec-as-source", makes it possible to write software which, in theory, treats **specs and design docs as the source code**, and treats the resulting agent-written code as something that has more in common with a build artifact than with source code.

## Setup for Copilot

1. Copy this AGENTS.md into your project
2. Copy these `.github/prompts/` and `.github/skills/` folders into your project's `.github/` folder
3. Restart your IDE
4. Now you can use the slash commands starting with `/spex-`

## Setup for Claude Code

1. Copy this AGENTS.md into your project
2. Copy these `.claude/commands/` and `.claude/skills/` folders into your project's `.claude/` folder
3. Restart your Claude Code environment
4. Now you can use the slash commands starting with `/spex:`
